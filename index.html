Run USB Drive On QEMU GUI

A Zenity-based launcher for booting physical USB drives, full raw disk images, and split boot/root image pairs in QEMU without rebuilding the command line by hand every time.

Open GitHub Repo Open Portfolio Page

This repo version is intentionally focused. It contains the launcher itself, a clean README, a license, and minimal repo scaffolding instead of unrelated rescue tooling.

At A Glance

Input Modes

Physical disk, full image, or separate boot and root images.

Disk Interface

virtio for speed or ide for compatibility.

First Run

Prompts for helper paths once, then stores them in user config.

Desktop Stack

Bash, Zenity, QEMU, OVMF, and virtiofsd.

Features

What The Launcher Improves

Less Repetitive QEMU Work

The tool compresses a repeated boot workflow into a guided interface instead of forcing the user to reconstruct a long command for each test run.

Cleaner Host Setup

Helper paths are detected when possible, prompted for when missing, and saved to a config file so the user does not need to re-enter them each launch.

Real Disk Support

It can target a physical USB drive directly, which is useful when testing installations or boot workflows against real media instead of only synthetic images.

Image Pair Support

It also handles the awkward case where the user has separate boot and root dumps instead of one clean full-disk image.

Quick Start

Getting It Running

Arch Package Baseline

sudo pacman -S qemu-desktop qemu-system-x86 edk2-ovmf virtiofsd zenity

Launch

chmod +x run-usb-drive-on-qemu-gui.sh
./run-usb-drive-on-qemu-gui.sh

On first run the launcher asks for the shared folder, the virtiofsd path, and the OVMF firmware paths if it cannot confirm them automatically. Those values are saved under ~/.config/run-usb-drive-on-qemu-gui/config.
